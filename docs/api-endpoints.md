# BookNest API Endpoint Guide

This document covers the API routes used by the current React frontend. The generated OpenAPI schema remains available from the backend, but this file records the practical contract the frontend depends on.

Base URL:

```text
http://localhost:8000/api/v1
```

Generated documentation:

- Swagger UI: `/api/v1/docs/`
- ReDoc: `/api/v1/redoc/`
- OpenAPI schema: `/api/v1/schema/`

Django admin:

- Admin site: `/admin/`
- Model-specific admin URLs are generated by Django Admin under `/admin/{app_label}/{model}/`.

## Shared Conventions

Authentication uses JWT bearer tokens:

```http
Authorization: Bearer <access-token>
```

Public catalog, search, author, genre, review, and catalog recommendation reads are available without a token unless the route notes otherwise. Mutations generally require authentication; admin catalog and operational routes require staff users.

Most list routes use page-number pagination:

```json
{
  "count": 42,
  "next": "http://localhost:8000/api/v1/books/?page=2",
  "previous": null,
  "results": []
}
```

Common pagination query params:

| Param | Type | Notes |
| --- | --- | --- |
| `page` | integer | 1-based page number. |
| `page_size` | integer | Clamped by `DRF_MAX_PAGE_SIZE`; default comes from `DRF_PAGE_SIZE`. |

Cursor-paginated routes, such as recommendations and feed events, return `next`, `previous`, and `results`; pass the `cursor` value from `next` to continue.

## Catalog Endpoints

### Books

`GET /books/`

Returns visible public books. Used by the frontend for catalog shelves, new releases, admin book lists, and fallback catalog browsing.

Query params:

| Param | Type | Notes |
| --- | --- | --- |
| `q` | string | Searches title, subtitle, authors, genres, description, ISBN-13, and ISBN-10. Must be blank or at least 2 characters. |
| `include_adult` | boolean | Defaults to `false`. |
| `genres` | comma string | Genre display names. |
| `genre_ids` | comma string | Genre IDs. |
| `authors` | comma string | Author display names. |
| `author_ids` | comma string | Author IDs. |
| `languages` | comma string | Language codes or labels stored on `Book.language`. |
| `sources` | comma string | Book source values. |
| `min_rating`, `max_rating` | number | 0 through 5. |
| `publication_year_from`, `publication_year_to` | integer | Inclusive publication-year range. |
| `page_count_min`, `page_count_max` | integer | Inclusive page-count range. |
| `is_featured` | boolean | Featured catalog filter. |
| `ordering` | enum | `trending`, `popular`, `rating`, `newest`, `title`, `featured`; default `trending`. |
| `page`, `page_size` | integer | Page-number pagination. |

Response item: `Book`.

Frontend consumers:

- `getCatalogBooks()` uses `/search/books/` for search/filter pages.
- `getNewReleaseBooks()` uses `/books/?ordering=newest`.
- `createBook()`, `updateBook()`, and `deleteBook()` use the write routes below.

## Search Endpoints

### Book Search

`GET /search/books/`

Searches the visible catalog, applies advanced filters, logs the search, and can queue external enrichment when there are no local results.

Query params:

| Param | Type | Notes |
| --- | --- | --- |
| `q` | string | Search term. Must be blank or at least 2 characters. |
| `genres`, `genre_ids` | comma string | Exact genre names or IDs. |
| `authors`, `author_ids` | comma string | Exact author names or IDs. |
| `languages`, `sources` | comma string | Book language/source filters. |
| `min_rating`, `max_rating` | number | 0 through 5. |
| `pub_date_from`, `pub_date_to` | date | Inclusive publication date range, `YYYY-MM-DD`. |
| `publication_year_from`, `publication_year_to` | integer | Inclusive year range. |
| `page_count_min`, `page_count_max` | integer | Inclusive page-count range. |
| `num_pages` | integer | Alias for minimum page count. |
| `include_adult` | boolean | Defaults to `false`. |
| `include_external` | boolean | If true and local results are empty, queues external enrichment if allowed. |
| `ordering` | enum | `relevance`, `trending`, `popular`, `rating`, `newest`, `title`; default `relevance`. |
| `page`, `page_size` | integer | Page-number pagination. |

Response is page-number paginated and includes search metadata:

```json
{
  "count": 12,
  "next": null,
  "previous": null,
  "results": [],
  "query": "dune",
  "filters_applied": {},
  "ordering": "relevance",
  "include_external": false
}
```

Frontend consumers:

- `searchBooks()` sends `q`, pagination, filters, `include_external`, and `ordering`.
- `useSearchBooks()` wires search result pages and search sorting.
- `getCatalogBooks()` wires Explore filters to the same endpoint for consistent paginated counts.

### Autocomplete And Suggestions

| Method | Path | Auth | Purpose |
| --- | --- | --- | --- |
| `GET` | `/search/autocomplete/` | Public | Paginated autocomplete terms. |
| `GET` | `/search/suggestions/` | Public | Suggestion envelope with `{ query, suggestions, count }`. |
| `GET` | `/search/related-books/` | Public | Suggested books from a `book_id` or `title`. |

Autocomplete query params:

| Param | Type | Notes |
| --- | --- | --- |
| `q` | string | Required, minimum 1 character. |
| `limit` | integer | 1 through 50; default 10. |
| `type` | enum | Optional term type: `book`, `author`, `genre`, `isbn`, or `query`. |

Autocomplete now falls back to live catalog matches when stored autocomplete terms do not fill the requested limit. The service persists those live terms for later requests.

Related suggestion query params:

| Param | Type | Notes |
| --- | --- | --- |
| `book_id` | integer | Reference book ID. |
| `title` | string | Alternative reference lookup when `book_id` is omitted. |
| `limit` | integer | 1 through 50; default 10. |
| `include_external` | boolean | Can queue enrichment when no local related books are found. |

Frontend consumers: `getSuggestions()`, `getSearchAutocomplete()`.

## Recommendations

### User Recommendations

| Method | Path | Auth | Purpose |
| --- | --- | --- | --- |
| `GET` | `/recommendations/` | Authenticated | Cursor-paginated active recommendations for the current user. |
| `GET` | `/recommendations/{id}/` | Authenticated | Current user's recommendation detail. |
| `POST` | `/recommendations/generate/` | Authenticated | Generate recommendations for the current user. |
| `POST` | `/recommendations/{id}/dismiss/` | Authenticated | Dismiss a recommendation. Returns `204`. |
| `POST` | `/recommendations/{id}/click/` | Authenticated | Mark a recommendation clicked and return it. |

Generate body:

```json
{
  "n_recommendations": 10,
  "model_id": null,
  "train_if_missing": true,
  "force_train": false,
  "async_generation": false
}
```

When `async_generation` is true, the endpoint returns `202` with a `RecommendationRun`. Otherwise it returns a list of `UserRecommendation` items. If training artifacts cannot be saved, generation falls back to non-model recommendations.

Frontend consumers: `getRecommendedBooks()`, `dismissRecommendation()`, `clickRecommendation()`.

When adding or changing an endpoint, update this document and the matching service function together. Keep API calls typed, normalize paginated responses through `client/src/lib/normalizers.ts`, and keep route-specific React Query keys in `client/src/features/catalog/hooks/catalog.keys.ts`.
